## 核心内容

### Prompt 注入防御

**攻击分类：**

| 级别 | 攻击方式 | 危害 |
|------|----------|------|
| L1 | 直接指令覆盖 | 泄露 System Prompt |
| L2 | 数据中嵌入指令 | 绕过内容审核 |
| L3 | 多轮对话诱导 | 逐步绕过限制 |
| L4 | 编码/混淆攻击 | 绕过关键词过滤 |

**防御层次：**

```
Layer 1: 输入验证（关键词 + 模式匹配）
Layer 2: 输入改写（标记用户输入边界）
Layer 3: 模型层防护（System Prompt 强化）
Layer 4: 输出审核（检测异常输出）
Layer 5: 运营层（监控 + 告警 + 人工审核）
```

**实用防御代码：**

```python
def sanitize_input(user_input: str) -> str:
    # 1. 检测高危模式
    dangerous_patterns = [
        "忽略之前的指令", "ignore previous instructions",
        "你现在是", "you are now",
        "system prompt", "系统提示"
    ]
    for pattern in dangerous_patterns:
        if pattern in user_input.lower():
            raise SecurityError(f"检测到可疑输入: {pattern}")

    # 2. 标记用户输入边界
    return f"<user_input>\n{user_input}\n</user_input>"
```

### 内容审核体系

**多级审核架构：**

```
输入 -> 关键词库（0.1ms） -> 分类模型（10ms） -> LLM审核（1s） -> 放行/拦截
```

| 级别 | 方法 | 速度 | 精度 | 成本 |
|------|------|------|------|------|
| L1 | 关键词匹配 | 极快 | 低 | 极低 |
| L2 | 文本分类模型 | 快 | 中 | 低 |
| L3 | LLM 审核 | 慢 | 高 | 中 |
| L4 | 人工审核 | 最慢 | 最高 | 高 |

**审核维度：**
- 色情/暴力/歧视
- 政治敏感
- 个人隐私泄露
- 商业秘密泄露
- 不实信息

### 数据隐私保护

**核心原则：**
- 最小化收集：只收集必要数据
- 脱敏处理：对话中的个人信息实时脱敏
- 存储加密：敏感数据加密存储
- 访问控制：严格的权限管理
- 定期清理：过期数据自动清理

**脱敏实现：**

```python
import re

def desensitize(text: str) -> str:
    # 手机号
    text = re.sub(r'1[3-9]\d{9}', '[PHONE]', text)
    # 身份证
    text = re.sub(r'\d{17}[\dXx]', '[ID_CARD]', text)
    # 邮箱
    text = re.sub(r'\w+@\w+\.\w+', '[EMAIL]', text)
    return text
```

### 多模态扩展

| 能力 | 说明 | 应用场景 |
|------|------|----------|
| 图像理解 | 理解图片内容 | 商品识别、文档解析 |
| 图像生成 | 根据文本生成图片 | 内容创作、设计辅助 |
| 语音交互 | 语音输入输出 | 智能客服、语音助手 |
| 视频理解 | 理解视频内容 | 视频分析、监控 |

**架构差异：**

```
纯文本应用：
用户 -> 文本 -> LLM -> 文本 -> 用户

多模态应用：
用户 -> 多模态输入 -> 编码器 -> 统一表示 -> LLM -> 解码器 -> 多模态输出 -> 用户
```

### 模型评估体系

**自动评估：**

| 指标 | 说明 | 适用任务 |
|------|------|----------|
| BLEU | n-gram 匹配度 | 翻译、摘要 |
| ROUGE | 召回率 | 摘要 |
| 准确率 | 分类正确率 | 分类 |
| 相似度 | Embedding 余弦相似度 | 语义匹配 |

**人工评估：**
- 相关性：回答是否切题
- 准确性：事实是否正确
- 完整性：是否回答了所有要点
- 流畅性：表达是否通顺
- 安全性：内容是否合规

**评估流程：**

```
1. 构建评估数据集（100-500条）
2. 自动评估跑基线
3. 人工抽检评估
4. 记录问题和改进方向
5. 迭代优化
```

### AI 原生产品设计思维

**与传统产品的区别：**

| 维度 | 传统产品 | AI 原生产品 |
|------|----------|------------|
| 交互 | 固定流程 | 自然语言对话 |
| 输出 | 确定性 | 概率性 |
| 体验 | 即时响应 | 流式生成 |
| 错误 | 可枚举 | 不可枚举 |
| 迭代 | 功能迭代 | 模型+Prompt迭代 |

**设计原则：**
- 设计"容错"体验，而非"零错"体验
- 给用户修正模型错误的能力
- 流式输出，不要让用户干等
- 错误兜底要有温度

## 重点认知

1. **安全是生产系统的底线**，不是加分项
2. **内容审核要分层**，不是一层过滤搞定
3. **评估要常态化**，不是上线前做一次
4. **AI 产品要接受不完美**，关键是快速修复

2. 实现输入输出的内容审核
3. 构建一个小型评估数据集
4. 做一次完整的安全审计
