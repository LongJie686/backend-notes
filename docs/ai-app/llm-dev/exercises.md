### 练习 1：构建一个多模型 API 调用封装

```python
class UnifiedLLMClient:

    def chat(self, provider_name, messages, stream=False, **kwargs):
        """统一调用接口"""
        pass

    def count_tokens(self, provider_name, text):
        """Token 计数"""
        pass
```

**考察点**：API 调用、抽象设计、错误处理、流式响应。

### 练习 2：实现一个完整的 RAG 系统

**目标**：从文档加载到回答生成，实现一个端到端的 RAG 系统。

**要求**：

1. 支持 PDF 和 Markdown 格式的文档加载
2. 实现递归切分策略（chunk_size=500, overlap=50）
3. 使用 Chroma 作为向量数据库
4. 实现混合检索（向量 + BM25）
5. 实现基本的重排序
6. 组装 Prompt 时包含检索结果和来源信息

**参考流程**：

```
文档 -> 加载 -> 切分 -> 向量化 -> 存储
查询 -> 向量检索 -> BM25检索 -> 混合融合 -> 重排序 -> Prompt组装 -> 生成
```

**考察点**：文档处理、切分策略、向量数据库、混合检索、重排序、Prompt 设计。

3. 设置最大迭代次数防止无限循环

## 思考题

### 基础原理

1. 如果让你从零解释 Transformer 给一个没有机器学习背景的开发者，你会怎么讲？尝试用生活中的类比来说明自注意力机制。

2. 为什么 Decoder-only 架构（如 GPT）在生成任务上比 Encoder-Decoder 架构更主流？它的优势在哪里？

3. 位置编码的作用是什么？如果不加位置编码，模型能正常工作吗？为什么？

### Prompt 工程

4. 设计一个客服场景的 Prompt 模板，要求：(1) 能处理退换货、物流查询、产品咨询三种意图；(2) 输出格式统一；(3) 对无法回答的问题有优雅的降级处理。

5. 你如何向团队证明一个新的 Prompt 比旧的更好？设计一个量化评估方案。

6. 如果模型的输出总是包含不需要的"废话"（如"好的，让我来帮你..."），你会怎么优化？给出至少三种方法。

### RAG 系统

7. 假设你的 RAG 系统检索到了 10 条相关文档，但上下文窗口只能放 5 条。你会怎么选择和排序？设计一个选择策略。

8. 一个 RAG 系统回答"我不知道"的比例过高，可能的原因有哪些？给出排查思路。

9. 如何判断 RAG 系统的问题出在检索环节还是生成环节？设计一个诊断方法。

10. 对于一个频繁更新的知识库（如产品文档），如何保证 RAG 系统的检索结果是最新的？

### Agent 设计

11. 设计一个旅行规划 Agent，它需要调用哪些工具？多 Agent 架构和单 Agent 架构哪种更适合？为什么？

12. Agent 的工具数量从 3 个增加到 20 个，可能会出现什么问题？如何解决？

13. 如何评估一个 Agent 的效果？需要哪些指标？和评估普通 LLM 应用有什么不同？

### 工程化

14. 你的大模型应用每天处理 10 万次请求，平均每次消耗 2000 Token（输入 1500 + 输出 500），使用 GPT-4o。请估算：(1) 每天的 API 费用；(2) 如果用户量增长 10 倍，费用怎么变化；(3) 给出至少 3 个降低成本的具体方案。

15. 设计一个大模型应用的灰度发布方案。需要考虑哪些因素？如何衡量灰度是否成功？

16. 流式响应中断（如网络波动导致 SSE 连接断开）如何处理？设计一个恢复机制。

### 安全与合规

17. 一个用户通过精心构造的输入，成功让模型输出了系统提示词。分析可能的攻击路径，设计防御方案。

18. 你的应用需要处理用户的医疗咨询数据。如何设计数据隐私保护方案，确保符合相关法规要求？

19. 内容审核系统误判率过高（正常内容被拦截），如何优化？给出调优思路。

### 架构设计

20. 设计一个支持多租户的大模型应用平台。需要考虑：模型路由、成本分摊、数据隔离、配额管理。画出架构图并说明关键设计决策。
