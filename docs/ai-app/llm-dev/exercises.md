# 练习题与思考

> 在开始学习之前或学完每个阶段后，可以用这些题目检验理解程度。

## 课前练习

### 练习 1

用你自己的话解释：大模型和传统的 if-else 规则引擎 / 关键词匹配系统，在"理解用户意图"这件事上，本质区别是什么？

**参考方向：**
- 传统系统基于规则匹配，能处理的意图是可枚举的
- 大模型基于语义理解，能处理未见过的表达方式
- 大模型有推理能力，能理解隐含意图
- 但大模型不够稳定，传统系统更可控

### 练习 2

如果你的老板说："我们有一堆内部文档，想让 AI 能回答关于这些文档的问题"，你会怎么设计这个系统的整体架构？画一个简单的架构图或文字描述。

**参考方向：**
- 文档管理模块：上传、解析、切分
- 知识库模块：Embedding、向量存储、索引管理
- 检索模块：查询改写、混合检索、重排序
- 生成模块：Prompt 模板、LLM 调用、流式输出
- 安全模块：输入审核、输出审核
- 运维模块：日志、监控、评估

### 练习 3

为什么"调用 OpenAI API + 写几行 Prompt" 不等于 "构建了一个大模型应用"？从工程角度列出至少 5 个需要额外解决的问题。

**参考方向：**
1. 成本控制（Token 用量管理、缓存策略）
2. 安全防护（Prompt 注入防御、内容审核）
3. 可靠性（错误处理、降级方案、重试机制）
4. 可观测性（日志、监控、调用链追踪）
5. 数据管理（对话历史存储、知识库维护）
6. 用户体验（流式输出、加载态、错误兜底）
7. 评估体系（效果评估、持续优化）
8. 部署运维（容器化、CI/CD、灰度发布）

## 阶段思考题

### 基础原理阶段

1. 如果 Transformer 的自注意力机制时间复杂度是 O(n^2)，当上下文窗口非常大时（如 1M Token），会有什么问题？目前有哪些解决方案？
2. 为什么说 Temperature = 0 时模型输出仍然不是完全确定性的？
3. BPE Tokenizer 对中文和英文的处理有什么区别？这对 Prompt 设计有什么影响？

### Prompt Engineering 阶段

4. 设计一个 Prompt，让模型从一段新闻中提取：人物、时间、地点、事件、影响。要求输出 JSON 格式。
5. 如果你的 Few-shot 示例之间有冲突（比如同一个词在不同示例中分类不同），模型会怎么表现？怎么避免？
6. 怎么量化评估两个 Prompt 版本哪个更好？

### RAG 阶段

7. 如果 RAG 系统的检索结果总是不相关，你会从哪些环节排查？
8. 为什么混合检索的效果通常比纯向量检索好？有没有例外？
9. 如果文档是表格数据（如 Excel），RAG 该怎么处理？

### Agent 阶段

10. 如果一个 Agent 在调用工具后始终无法得到满意的结果，你会怎么优化？
11. 多 Agent 协作中，Agent 之间怎么传递上下文才不会丢失关键信息？
12. 怎么设计一个 Agent 的"止损"机制，让它在卡住时能优雅退出？

### 工程化阶段

13. 如果 LLM API 的 P99 延迟突然从 2s 涨到 30s，你会怎么排查和应对？
14. 怎么设计一个成本友好的模型路由策略？
15. 如果你的大模型应用要支持 10 万 DAU，架构上要做什么准备？
