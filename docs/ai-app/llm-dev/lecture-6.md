# 第6讲：模型微调与优化

## 核心结论（5 条必记）

1. **微调不是万能药** -- 90% 的场景 Prompt + RAG 就够了，需要特定输出风格/格式或领域深度不够时才考虑微调
2. **数据质量是微调效果的决定因素** -- 不是模型大小，质量 > 数量，脏数据会让模型变差
3. **LoRA 是性价比最高的微调方式** -- 在原始权重旁加低秩分解矩阵，参数量减少 90%+ 但效果接近全量微调
4. **量化对推理几乎无影响** -- INT4 量化通常只损失 1-3% 精度，大胆用，能大幅降低显存和推理延迟
5. **微调 vs Prompt vs RAG 有明确决策框架** -- 知识更新用 RAG，行为定制用微调，通用任务用 Prompt

---

## 一、微调决策框架

### Prompt vs RAG vs 微调

| 维度 | Prompt | RAG | 微调 |
|------|--------|-----|------|
| 知识更新 | 实时 | 实时（更新库） | 需重训 |
| 成本 | 低 | 中 | 高 |
| 延迟 | 快 | 中 | 快 |
| 定制化程度 | 低 | 中 | 高 |
| 适用场景 | 通用任务 | 知识密集 | 行为定制 |

**什么时候需要微调：**
- 需要模型输出特定风格或格式
- 特定领域任务效果不够好
- 需要减少 Prompt 长度降低成本
- 需要模型学会特定推理模式

---

## 二、LoRA / QLoRA 原理与实践

### LoRA 核心原理

**核心思想：** 不修改原始模型权重，而是在旁边加一个低秩矩阵。

```
原始权重 W (d x d)  ->  冻结
LoRA 权重 A (d x r) 和 B (r x d)  ->  可训练

输出 = W * x + (A * B) * x
其中 r << d，参数量大幅减少
```

**LoRA Rank (r) 选择：**
- r = 4-8：简单任务（分类、格式化）
- r = 16-32：中等任务（对话、写作）
- r = 64-128：复杂任务（推理、代码）

**QLoRA：** 在 LoRA 基础上对基座模型做 4-bit 量化，进一步降低显存需求。

## 三、SFT 监督微调与 RLHF

### SFT 数据准备

**数据格式：**

```json
{
  "instruction": "请将以下文本翻译为英文",
  "input": "今天天气真好",
  "output": "The weather is really nice today."
}
```

**数据要求：**
- 数据量：几百到几万条，取决于任务复杂度
- 质量 > 数量：脏数据会让模型变差
- 多样性：覆盖各种输入模式
- 一致性：相同类型的输入应有相似的输出风格

**训练参数：**

| 参数 | 推荐值 | 说明 |
|------|--------|------|
| learning_rate | 1e-4 ~ 2e-5 | LoRA 可以稍大 |
| batch_size | 4-16 | 显存允许范围内尽量大 |
| epochs | 3-5 | 监控验证集 Loss |
| warmup_steps | 总步数的 10% | 避免初始学习率过大 |

### RLHF / DPO 对齐训练

**RLHF 流程：**
```
SFT 模型 -> 训练奖励模型 -> PPO 强化学习 -> 对齐模型
```

**DPO（Direct Preference Optimization）：**
- 不需要单独训练奖励模型
- 直接用偏好数据优化
- 流程更简单，效果接近 RLHF

## 四、模型量化与训练问题排查

### 模型量化

| 方法 | 精度 | 显存节省 | 精度损失 |
|------|------|----------|----------|
| FP16 | 16-bit | 50% | 几乎无 |
| INT8 | 8-bit | 75% | 小 |
| INT4 (GPTQ) | 4-bit | 87.5% | 中等 |
| INT4 (AWQ) | 4-bit | 87.5% | 较小 |

**量化选型建议：**
- 部署推理：INT4 量化即可满足大多数场景
- 追求精度：INT8 或 AWQ
- 原型验证：FP16

### 训练常见问题

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 不收敛 | 学习率过大、数据问题 | 降低学习率、检查数据质量 |
| 过拟合 | 数据太少、训练太久 | 早停、增加数据、降低 epochs |
| 灾难性遗忘 | 全量微调覆盖原始知识 | 用 LoRA、控制学习率 |
| 显存不足 | 模型或 batch 太大 | QLoRA、梯度累积、减小 batch |

## 五、面试高频题

### 1. LoRA 的原理是什么？为什么能用很少的参数？
在原始权重旁加低秩分解矩阵（A*B），只训练这两个小矩阵 -> 假设权重变化是低秩的，用很少参数就能表达 -> 参数量减少 90%+ 但效果接近全量微调

### 2. 微调数据要多少才够？
简单任务（格式化）50-200条，中等任务（对话风格）500-2000条，复杂任务（领域推理）2000+条 -> 质量远比数量重要 -> 数据量不够时优先提升数据质量

### 3. 量化会损失多少精度？
INT4 量化通常只损失 1-3% 的基准分数 -> 但对推理速度和显存节省巨大 -> AWQ 方法损失最小，推荐优先使用
